# O'z DSt 2815:2014 — Межсетевые экраны. Классификация

**Статус:** Действующий национальный стандарт
**Разработчик:** Служба национальной безопасности Республики Узбекистан
**Дата введения:** 31.03.2014

Данный стандарт устанавливает классификацию межсетевых экранов (МЭ) по уровню защищенности от несанкционированного доступа (НСД).

## 1. Основные понятия

- **Межсетевой Экран (МЭ):** Программное или программно-аппаратное средство, реализующее контроль за входящей и исходящей информацией.

- **Правила фильтрации:** Перечень условий, по которым разрешается или запрещается передача пакетов.

- **Трансляция адресов (NAT):** Функция МЭ, скрывающая внутренние адреса объектов от внешних субъектов.

## 2. Классы защищенности МЭ

Стандарт устанавливает **три класса** защищенности:

- **Класс 3:** Обеспечение безопасного взаимодействия АС, обрабатывающей конфиденциальную информацию, с внешней средой (Интернет).

- **Класс 2:** Обеспечение безопасного взаимодействия АС, обрабатывающей информацию с грифом «секретно».

- **Класс 1:** Обеспечение безопасного взаимодействия АС с информацией «совершенно секретно».

## 3. Требования (Таблица 1)

| Показатель защищенности | Класс 3 | Класс 2 | Класс 1 |
| :--- | :---: | :---: | :---: |
| Управление доступом (фильтрация / NAT) | + | + | + |
| Идентификация и аутентификация | - | + | + |
| Регистрация событий | + | + | + |
| Администрирование: Идентификация | + | + | + |
| Помехоустойчивость и целостность | + | + | + |
| Восстановление после сбоев | + | + | + |
| Тестирование | + | + | + |

## 4. Требования к Классу 3 (для АИС КРР)

Для систем типа АИС КРР, работающих с конфиденциальными данными, МЭ должен обеспечивать:

- **Фильтрацию на сетевом уровне:** По IP-адресам, портам и интерфейсам.

- **Регистрацию событий:** Запись времени, адреса и результата фильтрации в Audit Log.

- **Идентификацию администратора:** Защищенный вход для настройки правил.

- **Контроль целостности:** Проверка неизменности программного обеспечения самого МЭ.
