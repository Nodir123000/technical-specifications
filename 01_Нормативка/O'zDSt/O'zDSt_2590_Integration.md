# O'z DSt 2590:2012 — Требования к интеграции систем

Этот стандарт является ключом к созданию единого информационного пространства в Узбекистане. Без его соблюдения интеграция с Казначейством или ГНК будет невозможна.

## 🔗 Ключевые принципы интеграции

### 1. Сервис-ориентированная архитектура (SOA)
*   Взаимодействие должно строиться на основе **Web-сервисов** (REST/SOAP).
*   Система должна быть модульной и масштабируемой.

### 2. Унификация данных
*   Использование **единых государственных справочников и классификаторов** (Единый реестр справочников).
*   Форматы обмена данными должны быть стандартизированы (JSON/XML).

### 3. Гарантированная доставка и безопасность
*   Обеспечение целостности и подлинности передаваемой информации.
*   Однозначная идентификация участников обмена через защищенные ключи.

## 🗝 Применение в АИС КРР
В Разделе 4.4 нашего ТЗ необходимо прописать:
*   Использование API для обмена данными с внешними ведомствами.
*   Соответствие форматов обмена Единым технологическим требованиям.
