# O'z DSt 2816:2014 — ПО средств защиты информации. Контроль НДВ

**Статус:** Действующий национальный стандарт
**Разработчик:** Служба национальной безопасности Республики Узбекистан
**Дата введения:** 31.03.2014

Данный стандарт устанавливает классификацию программного обеспечения (ПО) средств защиты информации (СЗИ) по уровню контроля отсутствия в нем **недекларированных возможностей (НДВ)**.

## 1. Основные понятия

- **Недекларированные Возможности (НДВ):** Функциональные возможности ПО, не описанные в документации, которые могут привести к нарушению безопасности (конфиденциальности, целостности, доступности).

- **Программные закладки:** Преднамеренно внесенные объекты, инициирующие несанкционированные функции (backdoors).

- **Статический анализ:** Анализ исходных текстов без выполнения программы (ревью кода, декомпозиция).

- **Динамический анализ:** Контроль выполнения программы на реальных маршрутах и входных данных.

## 2. Уровни контроля

Устанавливается **три уровня** контроля отсутствия НДВ:

- **Уровень 3:** Минимальный. Предназначен для СЗИ, используемых в системах с конфиденциальной информацией.

- **Уровень 2:** Средний. Для систем с информацией под грифом «секретно».

- **Уровень 1:** Максимальный. Для систем с информацией под грифом «совершенно секретно».

## 4. Требования для АИС КРР

Поскольку АИС КРР обрабатывает конфиденциальную информацию и персональные данные, она должна соответствовать **Уровню 3** контроля НДВ. Это требует:

1. **Полной документации:** Наличие спецификаций на все модули СЗИ (RBAC, Audit Log, Encryption).

1. **Статического анализа:** Весь код модулей безопасности должен пройти ревью на отсутствие скрытых функций.

1. **Сертифицированных инструментов:** Использование SAAST/DAST инструментов (например, SonarQube, Snyk) для автоматизированного поиска уязвимостей и НДВ.
