# UC-03: Генерация и утверждение Акта ревизии

**ID:** UC-03
**Приоритет:** Критический
**Акторы:** Главный инспектор (CHIEF_INSPECTOR), Инспектор (INSPECTOR)
**Агенты:** Orchestrator, Security, Process, User
**Связанные тест-кейсы:** TC-ACT-001 — TC-ACT-012
**Зависит от:** UC-01, UC-02

---

## Precondition

* Ревизия имеет статус `IN_PROGRESS`

* Все нарушения зарегистрированы и проверены

* CHIEF_INSPECTOR назначен ответственным за ревизию

---

## Main Flow

```text

1. CHIEF_INSPECTOR → UI: Ревизия → "Сформировать Акт ревизии"

2. User Agent → Security Agent: Проверить права (CHIEF_INSPECTOR, CREATE report)

3. Security Agent: ✅ Разрешено (§З.3.6)

4. Process Agent → step-validator:

    Проверить: все нарушения зарегистрированы, данные полны?

1. step-validator → Process Agent: ✅ Условия выполнены

2. Process Agent → automated-reporting:

    {report_type: ACT_REVISION, audit_id, findings, locale: ru, output_format: BOTH}

1. automated-reporting: Генерирует XLSX + PDF по официальному шаблону

2. automated-reporting → Process Agent:

    {report_id, files: [{format:PDF, url:...}, {format:XLSX, url:...}]}

1. Process Agent → User Agent: Передать ссылки на файлы

2. User Agent → CHIEF_INSPECTOR: "Акт сформирован. Просмотрите и утвердите."

3. CHIEF_INSPECTOR → UI: Открыть PDF, проверить содержание

4. CHIEF_INSPECTOR → UI: Нажать "Утвердить" (Human-in-the-loop)

5. User Agent → Process Agent: Запрос смены статуса → Утверждён

6. Process Agent → step-validator:

    {current: "На проверке", target: "Утверждён", signed_by: chief_id}

1. step-validator → Process Agent: ✅ Переход разрешён

2. Process Agent → update-status: Статус ревизии → Утверждён

3. Process Agent → notify: Уведомить всех участников об утверждении

4. Process Agent → Security Agent: audit_log (ACT_APPROVED)

5. Monitoring Agent: Метрика audit_approved+1

```

---

## Alternative Flow A: Акт требует доработки

```text
12a. CHIEF_INSPECTOR → UI: Нажать "Вернуть на доработку"
12b. CHIEF_INSPECTOR → UI: Указать причину возврата
13a. Process Agent → step-validator: {target: "Черновик"}
15a. step-validator → Process Agent: ✅ Возврат разрешён
16a. Process Agent → update-status: Статус → Черновик
17a. Process Agent → notify: Уведомить INSPECTOR о необходимости доработки
```

## Alternative Flow B: Ошибка генерации PDF

```text
7b. automated-reporting: Ошибка рендеринга PDF
7c. Автоматический перезапуск с резервным рендерером
7d. Если снова ошибка: вернуть только XLSX
7e. User Agent → CHIEF_INSPECTOR: "Акт сформирован в формате XLSX. PDF временно недоступен."
```

---

## Postcondition

* Акт ревизии утверждён и хранится в защищённом хранилище

* Статус ревизии: `Утверждён`

* Все участники уведомлены

* Ревизия готова к отправке на Единый портал (UC-04)

---

## Human-in-the-loop точки

| Момент | Действие | Роль |
|:---|:---|:---|
| Просмотр сформированного акта | Рецензия содержания | CHIEF_INSPECTOR |
| Нажатие "Утвердить" | Финальное утверждение | CHIEF_INSPECTOR |
| При возврате на доработку | Указание причин | CHIEF_INSPECTOR |
