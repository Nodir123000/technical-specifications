# UC-01: Проведение ревизии (в закрытом контуре)

**ID:** UC-01
**Приоритет:** Критический (основной сценарий системы)
**Акторы:** Инспектор (INSPECTOR), Главный инспектор (CHIEF_INSPECTOR)
**Агенты:** Orchestrator, Security, Process, Integration, User
**Связанные тест-кейсы:** TC-AUDIT-001 — TC-AUDIT-015

---

## Precondition

* Пользователь аутентифицирован локально (логин/пароль + аппаратный токен/PIN)

* Организация-объект ревизии существует в системе

* Существует утверждённый план ревизии на текущий период

* Приказ о проведении ревизии утверждён CHIEF_INSPECTOR

---

## Main Flow (Основной путь)

```text

1. INSPECTOR → UI: Открыть раздел "Ревизии" → "Создать новую"

2. User Agent → Security Agent: Проверить права на CREATE для audits

3. Security Agent → User Agent: ✅ INSPECTOR: CREATE разрешён (§З.3.4)

4. User Agent → INSPECTOR: Отобразить форму создания ревизии

5. INSPECTOR → UI: Заполнить поля (организация, период, тип, состав комиссии)

6. User Agent → Process Agent: Передать данные формы

7. Process Agent → data-validation: Валидировать данные (schema: audit.create)

8. data-validation → Process Agent: ✅ Данные корректны

9. Process Agent → БД: Проверить наличие локально импортированных данных казначейства для этой организации и периода

10. Process Agent → User Agent: Сообщить о статусе данных ("Данные загружены" или "Требуется импорт")

11. Process Agent → БД: Создать запись ревизии (статус: Черновик)

12. Process Agent → Security Agent: Записать в audit_log (событие: AUDIT_CREATED)

13. User Agent → INSPECTOR: ✅ Ревизия создана. Перейти к загрузке Excel-данных или вводу нарушений?

```

---

## Alternative Flow A: Данные казначейства отсутствуют в локальной БД

```text
9a. Process Agent: Данные за указанный период отсутствуют
9b. Process Agent → User Agent: Предупреждение "Финансовые данные за период не обнаружены"
9c. User Agent → INSPECTOR: Предложить перейти к сценарию UC-04 (Ручной импорт Excel)
9d. INSPECTOR → UI: Выполнить импорт или продолжить создание ревизии (с пометкой о необходимости синхронизации)
```

## Alternative Flow B: Недостаточно прав

```text
2b. Security Agent → User Agent: ❌ Роль VIEWER не имеет права CREATE
2c. User Agent → VIEWER: "Недостаточно прав для выполнения операции"
→ Сценарий завершён
```

---

## Postcondition

* Ревизия создана в БД со статусом `Черновик`

* Лог-файл зафиксировал создание объекта в закрытом контуре

* Deadline-watchdog зарегистрировал дедлайн завершения ревизии

* Система готова к анализу загруженных Excel-данных
