# UC-01: Проведение ревизии

**ID:** UC-01
**Приоритет:** Критический (основной сценарий системы)
**Акторы:** Инспектор (INSPECTOR), Главный инспектор (CHIEF_INSPECTOR)
**Агенты:** Orchestrator, Security, Process, Integration, User
**Связанные тест-кейсы:** TC-AUDIT-001 — TC-AUDIT-015

---

## Precondition

- Пользователь аутентифицирован с ролью `INSPECTOR` или выше
- Организация-объект ревизии существует в системе
- Существует утверждённый план ревизии на текущий период
- Приказ о проведении ревизии утверждён CHIEF_INSPECTOR

---

## Main Flow (Основной путь)

```
1.  INSPECTOR → UI: Открыть раздел "Ревизии" → "Создать новую"
2.  User Agent → Security Agent: Проверить права на CREATE для audits
3.  Security Agent → User Agent: ✅ INSPECTOR: CREATE разрешён (§З.3.4)
4.  User Agent → INSPECTOR: Отобразить форму создания ревизии
5.  INSPECTOR → UI: Заполнить поля (организация, период, тип, состав комиссии)
6.  User Agent → Process Agent: Передать данные формы
7.  Process Agent → data-validation: Валидировать данные (schema: audit.create)
8.  data-validation → Process Agent: ✅ Данные корректны
9.  Process Agent → Integration Agent: Запросить финансовые данные организации
10. Integration Agent → TREASURY_MINFIN: fetch-treasury (период ревизии)
11. TREASURY_MINFIN → Integration Agent: Транзакции получены
12. Integration Agent → Process Agent: Данные переданы
13. Process Agent → БД: Создать запись ревизии (статус: Черновик)
14. Process Agent → Security Agent: Записать в audit_log (событие: AUDIT_CREATED)
15. Process Agent → Monitoring Agent: Зафиксировать метрику (audit_created+1)
16. User Agent → INSPECTOR: ✅ Ревизия создана. Перейти к вводу нарушений?
```

---

## Alternative Flow A: Данные казначейства недоступны

```
10a. Integration Agent → TREASURY_MINFIN: Ошибка 503
10b. Integration Agent: Retry × 3 с backoff
10c. Integration Agent → Process Agent: ❌ Система недоступна
10d. Process Agent → User Agent: Предупреждение "Данные казначейства временно недоступны"
10e. INSPECTOR → UI: Подтвердить создание без данных казначейства
10f. Process Agent → БД: Создать ревизию с флагом needs_treasury_sync=true
10g. Integration Agent: Поставить задачу синхронизации в очередь ретрая
```

## Alternative Flow B: Недостаточно прав

```
2b. Security Agent → User Agent: ❌ Роль VIEWER не имеет права CREATE
2c. User Agent → VIEWER: "Недостаточно прав для выполнения операции"
→ Сценарий завершён
```

---

## Postcondition

- Ревизия создана в БД со статусом `Черновик`
- Все члены комиссии получили уведомление о назначении
- Событие зафиксировано в audit_log
- Deadline-watchdog зарегистрировал дедлайн завершения ревизии

---

## Human-in-the-loop точки

| Момент | Действие | Роль |
|:---|:---|:---|
| Перед созданием | Проверка формы | INSPECTOR |
| Перед утверждением Акта | Рецензия и подпись | CHIEF_INSPECTOR |
| Перед отправкой на портал | Финальное подтверждение | CHIEF_INSPECTOR |
