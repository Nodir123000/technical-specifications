# UC-04: Интеграция с системой казначейства

**ID:** UC-04
**Приоритет:** Высокий
**Акторы:** Система (автоматический), Инспектор (INSPECTOR)
**Агенты:** Orchestrator, Security, Integration, Process
**Связанные тест-кейсы:** TC-INT-001 — TC-INT-008

---

## Precondition

- Ревизия создана (статус: `IN_PROGRESS`)
- Период ревизии определён
- Система казначейства доступна (проверяется Monitoring Agent)

---

## Main Flow (Автоматический запрос при создании ревизии)

```
1.  Process Agent → Orchestrator: Запрос данных казначейства для audit_id
2.  Orchestrator → Security Agent: Авторизовать исходящий запрос Integration Agent
3.  Security Agent: ✅ Разрешено (интеграционный запрос от системы)
4.  Orchestrator → Integration Agent: fetch-treasury
    {period: {from, to}, organization_id}
5.  Integration Agent → external-integration:
    {target: TREASURY_MINFIN, operation: fetch, payload: {...}}
6.  external-integration → TREASURY_MINFIN: SOAP-запрос через TLS 1.2
7.  TREASURY_MINFIN → external-integration: Транзакции в формате XML
8.  external-integration → Integration Agent: Данные получены (latency_ms: 1240)
9.  Integration Agent → data-validation:
    Валидировать полученные транзакции (schema: treasury.import)
10. data-validation → Integration Agent: ✅ Данные корректны
11. Integration Agent → transform-format: XML → внутренний JSON
12. Integration Agent → Process Agent: Транзакции переданы
13. Process Agent → БД: Сохранить транзакции, привязать к audit_id
14. Integration Agent → Security Agent: audit_log (TREASURY_DATA_FETCHED)
15. Monitoring Agent: Метрика integration.treasury.success+1
```

---

## Alternative Flow A: Казначейство недоступно

```
6a. TREASURY_MINFIN: 503 Service Unavailable
6b. external-integration: Retry 1 (backoff 1с)
6c. external-integration: Retry 2 (backoff 2с)
6d. external-integration: Retry 3 (backoff 4с) — всё ещё 503
6e. Integration Agent → retry-queue: Поставить в очередь (приоритет HIGH)
6f. Integration Agent → Process Agent: ❌ Недоступно, задача в очереди
6g. Process Agent → БД: Установить needs_treasury_sync=true
6h. Monitoring Agent: Метрика integration.treasury.failure+1, алерт WARNING
```

## Alternative Flow B: Данные не прошли валидацию

```
10b. data-validation → Integration Agent: ❌ errors: [{field: "amount", code: "INVALID_FORMAT"}]
10c. Integration Agent → Security Agent: audit_log (TREASURY_VALIDATION_FAILED, level: WARN)
10d. Integration Agent → Process Agent: Частичные данные с указанием ошибок
10e. Process Agent → User Agent: Предупреждение INSPECTOR об ошибках в данных
```

---

## Postcondition

- Транзакции организации сохранены в БД и привязаны к ревизии
- Integration_log содержит запись о запросе
- При успехе: `needs_treasury_sync=false`
- При неуспехе: задача в очереди ретрая, инспектор уведомлён
