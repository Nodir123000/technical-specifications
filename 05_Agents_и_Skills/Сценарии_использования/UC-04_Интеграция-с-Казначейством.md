# UC-04: Синхронизация данных с казначейством (Локальный импорт)

**ID:** UC-04
**Приоритет:** Высокий
**Акторы:** Инспектор (INSPECTOR), Администратор (ADMIN)
**Агенты:** Orchestrator, Security, Integration, Process
**Связанные тест-кейсы:** TC-IM-01 — TC-IM-03 (Тесты импорта)

---

## Precondition

* Ревизия создана (статус: `IN_PROGRESS`)

* Инспектор получил выгрузку из внешней системы казначейства в формате Excel (.xlsx)

* Носитель (USB/CD) подключен к защищенному АРМ и проверен на вирусы

---

## Main Flow (Ручной импорт данных в закрытом контуре)

```text

1. INSPECTOR → User Agent: Загрузить файл выписки казначейства (audit_id)

2. User Agent → Orchestrator: Инициировать импорт (file_path, checksum)

3. Orchestrator → Security Agent: Проверить файл на целостность и безопасность

4. Security Agent: ✅ Файл проверен, вредоносного ПО не обнаружено, SHA-256 совпадает

5. Orchestrator → Integration Agent: local-excel-import (template: TREASURY_V1)

6. Integration Agent → local-excel-import skill:

    Считать данные из файла, проверить структуру колонок и типы данных

1. local-excel-import → Integration Agent: ✅ Данные успешно прочитаны (JSON)

2. Integration Agent → data-validation:

    Валидировать бизнес-логику транзакций (period, sums)

1. data-validation → Integration Agent: ✅ Данные корректны

2. Integration Agent → Process Agent: Транзакции переданы для сохранения

3. Process Agent → БД: Сохранить транзакции, привязать к audit_id

4. Integration Agent → Security Agent: audit_log (LOCAL_TREASURY_IMPORT_SUCCESS)

5. User Agent → INSPECTOR: "Данные казначейства успешно загружены: 145 записей"

```

---

## Alternative Flow A: Ошибка целостности файла

```text
4a. Security Agent: ❌ Контрольная сумма файла не совпадает (SHA-256 mismatch)
4b. Security Agent → Orchestrator: Блокировка операции импорта
4c. Orchestrator → Security Agent: audit_log (IMPORT_INTEGRITY_VIOLATION, level: CRITICAL)
4d. User Agent → INSPECTOR: "Ошибка: Файл поврежден или изменен после выгрузки"
```

## Alternative Flow B: Несовпадение шаблона Excel

```text
7a. local-excel-import: ❌ Ошибка (Отсутствует обязательная колонка "Назначение платежа")
7b. local-excel-import → Integration Agent: Error (TEMPLATE_MISMATCH)
7c. User Agent → INSPECTOR: "Ошибка: Файл не соответствует формату Казначейства. Используйте шаблон №2"
```

---

## Postcondition

* Финансовые данные казначейства загружены в локальную БД и доступны для анализа

* Лог-файл содержит запись об импорте с указанием контрольной суммы и исполнителя

* Риск работы с неактуальными данными (из-за отсутствия сети) минимизирован ручным импортом
