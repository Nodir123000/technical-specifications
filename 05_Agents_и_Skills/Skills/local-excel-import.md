# 📊 Skill: local-excel-import

**ID:** `local-excel-import`
**Версия:** 1.0
**Владелец:** Integration Agent
**Стандарт:** O'z DSt 1987:2018 §6.5.4 — Локальная синхронизация данных

---

## Описание

Управляет безопасным импортом данных из файлов Excel (.xlsx) в условиях закрытого сетевого контура. Обеспечивает проверку структуры, типов данных и целостности файлов перед передачей информации во внутренние модули системы.

---

## Поддерживаемые шаблоны импорта

| Источник данных | Код шаблона | Обязательные поля |
|:---|:---|:---|
| Казначейство (Выписки) | `TREASURY_V1` | Номер транзакции, Дата, Сумма, ИНН получателя, Назначение |
| Налоговая (Контрагенты) | `TAX_V1` | ИНН, Наименование, ОКОНХ, Статус плательщика НДС |
| Кадастр (Объекты) | `CADASTRE_V1` | Кадастровый номер, Адрес, Площадь, Владелец |

---

## Входные параметры

```json
{
  "file_path": "string (локальный путь к смонтированному файлу)",
  "template_id": "enum: [TREASURY_V1, TAX_V1, CADASTRE_V1]",
  "checksum_sha256": "string (ожидаемая контрольная сумма)",
  "user_id": "string (инициатор загрузки)",
  "security_scan_status": "boolean (подтверждение от Security Agent)"
}
```

---

## Выходные параметры

```json
{
  "success": "boolean",
  "rows_processed": "number",
  "rows_failed": "number",
  "error_log": "array (список строк с ошибками валидации)",
  "data_json": "object (преобразованные данные для Process Agent)",
  "import_timestamp": "datetime"
}
```

## SLA и Безопасность

| Метрика | Значение |
|:---|:---|
| Максимальный размер файла | 50 МБ |
| Время обработки (10 000 строк) | < 30 секунд |
| Глубина валидации | 100% строк и ячеек |
| Требование к носителю | Предварительная проверка Security Agent |

---

## Алгоритм работы

1. **Mount Check**: Проверка доступности файла по указанному пути.

1. **Security Gate**: Проверка флага `security_scan_status`. Если `false` — блокировка.

1. **Integrity Check**: Вычисление SHA-256 файла и сверка с входным параметром.

1. **Structure Validation**: Проверка наличия всех обязательных колонок согласно `template_id`.

1. **Type Validation**: Проверка типов данных в ячейках (даты, числа, ИНН).

1. **Transform**: Преобразование валидных строк во внутренний формат JSON.

1. **Handover**: Передача данных Process Agent и запись в лог импорта.

---

## Обработка ошибок

| Ошибка | Действие |
|:---|:---|
| Файл не найден | Сообщение пользователю, запись в аудит безопасности |
| НесовпадениеSHA-256 | Блокировкаимпорта,подозрениенаподменуданных |
| Ошибка структуры (колонки) | Прерывание, возврат списка недостающих колонок |
| Ошибки в данных (строки) | Пропуск ошибочных строк, формирование `error_log` для пользователя |

## Тест-кейсы

### TC-IM-01: Успешный импорт Казначейства

**Вход:** Файл TREASURY_V1 (100 строк), верная контрольная сумма.
**Ожидание:** 100 строк обработано, `success: true`, данные переданы в Process Agent.

### TC-IM-02: Ошибка целостности

**Имитация:** Измененный файл с неверным хэшем.
**Ожидание:** `success: false`, ошибка `CHECKSUM_MISMATCH`, запись в аудит.

### TC-IM-03: Частичный импорт

**Вход:** Файл, где 5 строк из 100 содержат текст вместо чисел в поле суммы.
**Ожидание:** 95 строк загружено, 5 строк в `error_log`, статус `PARTIAL_SUCCESS`.
