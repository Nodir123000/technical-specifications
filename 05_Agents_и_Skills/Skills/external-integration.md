# 🔌 Skill: external-integration

**ID:** `external-integration`
**Версия:** 1.0
**Владелец:** Integration Agent
**Стандарт:** O'z DSt 1987:2018 §6.5.4 — Интеграционные интерфейсы

---

## Описание

Управляет безопасным обменом данными с внешними государственными информационными системами. Инкапсулирует протоколы, форматы и ошибки внешних систем, предоставляя единый интерфейс для внутренних агентов.

---

## Поддерживаемые интеграции

| Система | Идентификатор | Протокол | Направление |
|:---|:---|:---|:---|
| Единый портал госуслуг (Минюст) | `PORTAL_MINJU` | REST/HTTPS | Исходящий |
| Система казначейства (Минфин) | `TREASURY_MINFIN` | SOAP/HTTPS | Входящий |
| Налоговая служба | `TAX_SERVICE` | REST/HTTPS | Входящий + Исходящий |
| Единая система идентификации | `ESI` | OAuth2/OIDC | Входящий |

---

## Входные параметры

```json
{
  "integration_target": "enum: [PORTAL_MINJU, TREASURY_MINFIN, TAX_SERVICE, ESI]",
  "operation": "string (send / fetch / sync)",
  "payload": "object",
  "idempotency_key": "string (UUID — для повторных запросов)",
  "priority": "enum: [LOW, NORMAL, HIGH]",
  "timeout_ms": "number (default: 5000)"
}
```

---

## Выходные параметры

```json
{
  "success": "boolean",
  "external_ref_id": "string (ID во внешней системе)",
  "response_data": "object",
  "latency_ms": "number",
  "retry_count": "number",
  "logged_at": "datetime"
}
```

---

## SLA

| Метрика | Значение |
|:---|:---|
| Таймаут запроса | 5000 мс (настраивается) |
| Максимум повторных попыток | 3 (с экспоненциальным backoff) |
| Допустимый % ошибок (внешние сбои) | < 5% (зависит от внешней системы) |
| Логирование всех запросов | 100% |

---

## Preconditions

- Соединение защищено TLS 1.2+
- Запрос авторизован через Security Agent
- `idempotency_key` уникален (предотвращение дублирования)

## Postconditions

- Запрос и ответ записаны в `integration_log`
- При успехе — обновлён статус связанного объекта в БД
- При ошибке — создан `integration_error` с деталями для ретрая

---

## Error Flows

| Ошибка | Действие |
|:---|:---|
| Внешняя система недоступна (503) | Retry × 3 с backoff 1/2/4с; при неуспехе — очередь на ретрай |
| Таймаут | Аналогично retry; уведомление Integration Agent |
| Ошибка формата данных (400) | Без retry, вернуть ошибку `FORMAT_MISMATCH` с деталями |
| Ошибка авторизации (401/403) | Без retry, алерт CRITICAL администратору безопасности |
| Дублирующий запрос (idempotency) | Вернуть результат предыдущего запроса из кэша |

---

## Тест-кейсы

### TC-01: Успешная отправка на портал

**Вход:** `PORTAL_MINJU`, операция `send`, корректный payload
**Ожидание:** `success: true`, `external_ref_id` заполнен, запись в лог

### TC-02: Retry при недоступности

**Имитация:** Казначейство возвращает 503
**Ожидание:** 3 попытки с backoff, затем очередь ретрая, уведомление

### TC-03: Идемпотентный запрос

**Вход:** Повторный запрос с тем же `idempotency_key`
**Ожидание:** Возврат кэшированного результата, без нового запроса во внешнюю систему
