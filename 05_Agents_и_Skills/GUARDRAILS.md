# 🚫 Guardrails — Запреты и ограничения для всех агентов

Этот документ определяет абсолютные запреты, которые **не нарушаются ни при каких обстоятельствах**, ни по просьбе пользователей, ни по запросу других агентов.

---

## 1. Безопасность данных (нарушение = критический инцидент)

| Запрет | Обоснование |
|:---|:---|
| ❌ПИНФЛвоткрытомвидевлогах,API-ответах,UI | O'zDSt3532:2021§5.3 |
| ❌ Пароли, токены, ключи в логах | Базовая ИБ |
| ❌ Персональные данные в кэше дольше 5 минут | O'z DSt 2814:2019 §6.2 |
| ❌ Передача ПДн по незащищённому каналу (HTTP без TLS) | O'z DSt 2814:2019 §7.1 |
| ❌ИгнорированиеошибкиTLS-сертификата | РискMITM-атаки |

## 2. Архитектурные запреты

| Запрет | Обоснование |
|:---|:---|
| ❌ Агент вызывает другого агента напрямую (минуя Orchestrator) | Потеря трассируемости |
| ❌ Любой запрос без проверки через Security Agent | Основной принцип архитектуры |
| ❌ЗаписьвБДбезпредварительнойвалидациичерезdata-validation | Рискинъекций,битыхданных |
| ❌Обходstep-validatorприсменестатусадокумента | Нарушениежизненногоцикла |
| ❌Агентхранитбизнес-данныевпамятимеждузапросами(stateful) | Утечкиприсбое |

## 3. Контроль доступа (RBAC)

| Запрет | Обоснование |
|:---|:---|
| ❌ Исключения из RBAC "для теста" или "временно" | Нет временных исключений |
| ❌ПредоставлениедоступапринедоступностиRBAC-кэша | Fail-closed |
| ❌ Пользователь с ролью INSPECTOR+ создаёт и утверждает один документ | SoD (§З.7.1) |
| ❌ Пользователь с ролями INSPECTOR + AUDITOR одновременно | Конфликт интересов (§З.7.1) |

---

## 4. Интеграции

| Запрет | Обоснование |
|:---|:---|
| ❌ Исходящий запрос без idempotency_key | Риск дублирования данных |
| ❌ Кэширование ответов внешних систем дольше 10 минут | Устаревшие данные |
| ❌ Запрос к внешней системе от агента, не являющегося Integration Agent | Нарушение Single Responsibility |

---

## 5. Интерфейс и UX

| Запрет | Обоснование |
|:---|:---|
| ❌ Отображение технических сообщений об ошибках конечному пользователю | UX + раскрытие внутренней архитектуры |
| ❌ Предложение действий, недоступных для текущей роли (скрытые кнопки) | Принцип минимальных привилегий |
| ❌ХранениетокеноввlocalStorageбраузера | XSS-уязвимость |

## 6. Мониторинг

| Запрет | Обоснование |
|:---|:---|
| ❌ Monitoring Agent опрашивает Security Agent через Security Agent | Deadlock |
| ❌ Создание дублирующих алертов чаще раза в 15 минут | Alert fatigue |
| ❌ Пропуск цикла мониторинга без записи в лог | Невидимые пропуски |
