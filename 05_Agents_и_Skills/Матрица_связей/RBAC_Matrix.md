# ПРИЛОЖЕНИЕ З — МАТРИЦА РАЗРЕШЕНИЙ (RBAC)

Автоматизированная Информационная Система Контрольно-Ревизионного Управления
Министерства Обороны Республики Узбекистан (АИС КРУ МО РУЗ)

---

## З.1. ОБЩИЕ ПОЛОЖЕНИЯ

### З.1.1. Назначение

Настоящее приложение определяет ролевую модель доступа (RBAC) для АИС КРУ МО РУЗ, включая перечень ролей, разрешений и матрицу «Роль × Операция».

### З.1.2. Принципы RBAC

1. **Принцип минимальных привилегий** — пользователь получает только необходимые права

1. **Разделение обязанностей** — критические операции требуют нескольких ролей

1. **Аудируемость** — все действия регистрируются в журнале аудита

## З.2. ПЕРЕЧЕНЬ РОЛЕЙ

### З.2.1. Основные роли

| Код роли | Наименование | Описание | Уровень доступа |
|----------|------------|---------|----------------|
| **ADMIN** | Администратор | Полный доступ к системе | Максимальный |
| **CHIEF_INSPECTOR** | Главный инспектор | Утверждение планов, аналитика | Высокий |
| **INSPECTOR** | Инспектор-ревизор | Планирование,проведениеревизий | Средний |
| **VIEWER** | Наблюдатель | Только чтение данных | Минимальный |

| Код роли | Наименование | Описание | Применение |

## З.5. ПРОЦЕСС НАЗНАЧЕНИЯ РОЛЕЙ

### З.5.1. Процедура назначения роли

| Этап | Действие | Ответственный | Результат |
|------|---------|--------------|----------|
| 1 | Заявка на назначение роли | Руководитель подразделения | Заполненная форма |
| 2 | Проверка обоснованности | Администратор | Заключение |
| 3 | Согласование с безопасностью | Администратор безопасности | Согласование/Отказ |
| 4 | Назначение роли | Администратор | Роль присвоена |
| 5 | Уведомление пользователя | Система | Email с данными доступа |
| 6 | Запись в журнал | Система | audit_log |

### З.5.2. Чек-лист при назначении роли

* [ ] Заявка заполнена и подписана руководителем

* [ ] Указаны: ФИО, должность, подразделение

* [ ] Указана запрашиваемая роль

* [ ] Обоснование необходимости роли

* [ ] Проверка отсутствия конфликта интересов

* [ ] Согласование с администратором безопасности

* [ ] Приказ о назначении (для ADMIN/SYSTEM_ADMIN)

## З.7. ОГРАНИЧЕНИЯ И ЗАПРЕТЫ

### З.7.1. Разделение обязанностей (SoD)

| Комбинация ролей | Запрет | Обоснование |
|-----------------|--------|------------|
| INSPECTOR + AUDITOR | Запрещено | Конфликт интересов |
| Создатель ревизии + Утверждающий | Запрещено | Проверка должна быть независимой |
| Финансовый + Проверяющий той же части | Ограничено | Только с согласования |

### З.7.2. Временные ограничения

| Ограничение | Значение | Применение |
|------------|---------|-----------|
| Максимум активных сессий | 3 | Все роли |
| Таймаут сессии | 30 мин | Все роли |
| Максимум операций в минуту | 100 | API endpoints |
| Период действия роли | 1 год | Все роли (кроме ADMIN) |
