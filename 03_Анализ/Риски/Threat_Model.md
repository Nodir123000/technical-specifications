## Ё.1. ОБЩИЕ ПОЛОЖЕНИЯ

### Ё.1.1. Назначение

Настоящее приложение определяет модель угроз информационной безопасности АИС КРУ МО РУЗ, включая классификацию угроз по модели STRIDE, оценку рисков, план защитных мер и требования к классу защиты.

### Ё.1.2. Класс защиты

| Параметр | Значение | Обоснование |
|----------|---------|------------|
| **Класс защиты** | **Класс 2** | ГосИС, обрабатывающая конфиденциальную информацию |
| **Нормативный документ** | O'zDSt 2814:2014 | Классификация АС по уровню защищённости |
| **Требуемый уровень защиты** | 99.9% | Высокий уровень критичности |

### Ё.1.3а. OWASP Top 10 (актуальные угрозы для веб-приложений)

| OWASP ID | Категория | Применимость к АИС КРУ |
|----------|-----------|------------------------|
| **A1** | Broken Access Control (Нарушение контроля доступа) | Высокая — RBAC должен строго проверяться на бэкенде |
| **A2** | Cryptographic Failures (Криптографические сбои) | Высокая — хранение ПДн, ПИНФЛ |
| **A3** | Injection (Инъекции) | Критическая — SQL-инъекции, XSS, CSRF |
| **A4** | Insecure Design (Небезопасное проектирование) | Средняя — требуется threat modeling |
| **A5** | Security Misconfiguration (Неправильная конфигурация) | Высокая — TLS, заголовки, WAF |
| **A6** | Vulnerable Components (Уязвимые компоненты) | Средняя — регулярный SAST/DAST |
| **A7** | Authentication Failures (Ошибки аутентификации) | Критическая — 2FA для админов |
| **A8** | Software and Data Integrity Failures | Средняя — CI/CD integrity |
| **A9** | Security Logging Failures | Высокая — 100% аудит |
| **A10** | SSRF (Server-Side Request Forgery) | Низкая — ограниченное использование |

> [!IMPORTANT]
> Модель угроз должна включать как минимум A1, A3, A7 (отмечены как критические). Для всех остальных категорий необходимо провести оценку риска.

## Ё.2. МОДЕЛЬ УГРОЗ STRIDE

### Ё.2.1. Классификация угроз по STRIDE

| Категория | Описание | Примеры угроз | Уровень риска |
|-----------|---------|--------------|--------------|
| **S** — Spoofing | Подмена идентичности | Кража учётных данных, фишинг | Высокий |
| **T** — Tampering | Изменение данных | SQL-инъекция, подделка документов | Критический |
| **R** — Repudiation | Отказ от действий | Удаление записей без логирования | Средний |
| **I** — Information Disclosure | Раскрытие информации | Утечка персональных данных | Критический |
| **D** — Denial of Service | Отказ в обслуживании | DDoS, перегрузка системы | Средний |
| **E** — Elevation of Privilege | Повышение привилегий | Взлом учётки, privilege escalation | Критический |

### Ё.2.2. Детальный перечень угроз

#### Ё.2.2.1. S — Spoofing (Подмена идентичности)

| № | Угроза | Описание | Источник | Вероятность | Влияние | Риск |
|---|--------|---------|---------|------------|---------|------|
| S-01 | Фишинг учётных данных | Злоумышленник отправляет поддельное письмо | Внешний | Средняя | Высокое | **Высокий** |
| S-02 | Кейлоггер | Вредоносное ПО перехватывает пароли | Внешний/Внутренний | Низкая | Высокое | **Средний** |
| S-03 | Подбор пароля | Брутфорс атака на учётные записи | Внешний | Средняя | Высокое | **Высокий** |
| S-04 | Перехват сессии | Атака «человек посередине» (MitM) | Внешний | Низкая | Высокое | **Средний** |
| S-05 | Социальная инженерия | Манипуляция пользователем для получения данных | Внешний | Средняя | Среднее | **Средний** |

#### Ё.2.2.2. T — Tampering (Изменение данных)

| № | Угроза | Описание | Источник | Вероятность | Влияние | Риск |

| T-02a | CSRF-атака (OWASP A3) | Подделка межсайтовых запросов | Внешний | Средняя | Среднее | **Средний** |
| T-03 | Подделка данных ревизии | Изменение акта проверки | Внутренний | Низкая | Критическое | **Высокий** |
| T-04 | Изменение сумм нарушений | Корректировка финансовых данных | Внутренний | Низкая | Критическое | **Высокий** |
| T-05 | Удаление записей аудита | Несанкционированное удаление логов | Внутренний | Низкая | Высокое | **Средний** |
| T-06 | Вредоносное ПО | Загрузка заражённого файла | Внешний | Средняя | Критическое | **Высокий** |

## Ё.7. ШИФРОВАНИЕ И КРИПТОГРАФИЯ

### Ё.7.1. Алгоритмы шифрования

| Назначение | Алгоритм | Ключ | Примечание |
|-----------|----------|------|----------|
| **Симметричное (at rest)** | AES-256-GCM | 256 бит | Данные в БД |
| **Асимметричное** | RSA-4096 | 4096 бит | ЭЦП, ключи обмена |
| **Хеширование паролей** | bcrypt | cost factor 12 | |
| **HMAC** | SHA-256 | 256 бит | Целостность сообщений |
| **TLS** | TLS 1.3 | - | Каналы связи |

### Ё.7.2. Управление ключами

| Аспект | Требование | Реализация |
|--------|-----------|-----------|
| Генерация | Аппаратный RNG | HSM/TPM |
| Хранение | Отдельное хранилище | HashiCorp Vault |
| Ротация | Ежегодно или при компрометации | Vault Transit |
| Уничтожение | Невосстановимое удаление | Критерий NIST SP 800-88 |
