# ПРИЛОЖЕНИЕ Ё — МОДЕЛЬ УГРОЗ И ПЛАН БЕЗОПАСНОСТИ

**Редакция:** Поддержка закрытого сетевого контура (Air-Gapped)
**Стандарт:** O'zDSt 2814:2014; O'z DSt 3532:2021

---

## Ё.1. ОБЩИЕ ПОЛОЖЕНИЯ

### Ё.1.1. Назначение

Настоящее приложение определяет модель угроз информационной безопасности АИС КРУ МО РУЗ в условиях **физически изолированной локальной сети (Air-Gap)**. Отсутствие прямого подключения к сети Интернет переносит основной вектор угроз на физические носители и инсайдерские действия.

### Ё.1.2. Класс защиты и Среда

| Параметр | Значение | Обоснование |
| ---------- | --------- | ------------ |
| **Класс защиты** | **Класс 2** | ГосИС, обрабатывающая конфиденциальную информацию |
| **Сетевая среда** | Air-Gapped / Off-line | Физическая изоляция от публичных сетей |
| **Аутентификация** | Локальная 2FA | PIN-коды + Аппаратные токены |

## Ё.4. ПЛАН ЗАЩИТНЫХ МЕР (ОБНОВЛЕННЫЙ)

### Ё.4.1. Защита при импорте данных

1. **АРМ-фильтр (Air-Gap Gateway):** Все входящие носители проходят проверку на выделенном АРМ без ОС (Live USB) с 3+ антивирусными базами.
2. **Контроль целостности (SHA-256):** Импорт файла в АИС КРУ невозможен без ввода контрольной суммы, сгенерированной на станции выгрузки.
3. **Аудит портов:** На серверах системы программно заблокированы все порты, кроме утвержденных для синхронизации. Каждое подключение носителя логируется с указанием серийного номера.

### Ё.4.2. Локальная аутентификация

- **Исключение OneID:** Аутентификация переведена на внутреннюю службу `security-agent`.
- **Локальный 2FA:** Вход инспектора требует ввода пароля и 4-значного PIN-кода, привязанного к физическому рабочему месту или токену.

## Ё.7. ШИФРОВАНИЕ И КРИПТОГРАФИЯ (At Rest)

| Назначение | Алгоритм | Обоснование |
| ----------- | ---------- | ------------ |
| **Данные в БД** | AES-256-GCM | Защита от прямого копирования файлов базы |
| **Хеширование** | Argon2 / bcrypt | Защита локальных паролей |
| **Целостность файлов** | SHA-256 | Валидация Excel-шаблонов перед импортом |
