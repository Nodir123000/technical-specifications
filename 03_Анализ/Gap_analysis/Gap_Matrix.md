# Gap-анализ: "Как есть" → "Как будет"

**Назначение:** Определить разрыв между текущими процессами КРУ МО РУЗ и автоматизированными функциями АИС КРУ.

**Методология:** Каждое требование из ТЗ сопоставлено с текущим ручным/бумажным процессом.

---

## Матрица Gap-анализа

| ID | Требование | Процесс "as is" (сейчас) | Процесс "to be" (после внедрения) | Gap / Автоматизируется |
|:---|:---|:---|:---|:---|
| **ФТ-01** | Анализ транзакций на нарушения | Ручная проверка документов инспектором (1–3 дня на организацию) | Автоматический анализ `kru-inspection-expert` (≤ 8 сек) | ✅ Полная автоматизация |
| **ФТ-02** | Генерация Акта ревизии | Word/Excel вручную (4–8 часов), возможны ошибки форматирования | Автогенерация по шаблону `automated-reporting` (≤ 30 сек) | ✅ Полная автоматизация |
| **ФТ-03** | Контроль дедлайнов | Личный календарь инспектора, Excel-таблицы | `deadline-watchdog` — автоуведомления T-7/T-3/T-0 | ✅ Полная автоматизация |
| **ФТ-04** | Три языка интерфейса | Документы готовятся отдельно на каждом языке | `i18n-localization` — мгновенное переключение | ✅ Полная автоматизация |
| **ФТ-05** | Статусы документа | Отслеживание статуса по папкам/журналам вручную | `step-validator` — автоматический контроль переходов | ✅ Полная автоматизация |
| **ФТ-07** | Данные из казначейства | Запрос официальным письмом (3–7 рабочих дней) | `external-integration` — автозапрос при создании ревизии (≤ 8 сек) | ✅ Полная автоматизация |
| **ТБ-01** | RBAC | Ограничение доступа через папки/пароли на файлы | Централизованный RBAC через `access-control` | ✅ Полная автоматизация |
| **ТБ-02** | Маскирование ПИНФЛ | ПИНФЛ доступен всем, кто имеет доступ к файлу | Автомаскирование `security-guardian` по роли | ✅ Полная автоматизация |
| **ТБ-03** | Журнал аудита | Бумажный журнал входа-выхода | Цифровой `audit_log` с 100% покрытием | ✅ Полная автоматизация |

---

## Функции без автоматизации (остаются ручными)

| Функция | Обоснование оставить ручной |
|:---|:---|
| Подписание Акта ревизии | Требует ЭЦП и юридической ответственности должностного лица |
| Принятие решения о нарушении | Human-in-the-loop: главный инспектор несёт ответственность |
| Отправка актов на портал | Требует подтверждения уполномоченного лица |
