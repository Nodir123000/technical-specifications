## ПС-01: Планирование КРД (Контрольно-ревизионной деятельности)

| Атрибут | Значение |
|:---|:---|
| **Агент-владелец** | Process Agent |
| **Стандарт** | O'z DSt 1987:2018 §6.5.1 |

**Функции:**
- Формирование годовых, квартальных, месячных планов ревизий
- Согласование и утверждение планов (workflow)
- Создание приказов на проведение ревизий
- Контроль исполнения плановых показателей

**Входные данные:** Директивы Министерства, данные организаций-объектов
**Выходные данные:** Утверждённые планы, приказы, назначения инспекторов
**Связанные подсистемы:** ПС-02 (передаёт задания), ПС-06 (контроль прав)

---

## ПС-02: Управление ревизиями

| Атрибут | Значение |
|:---|:---|
| **Агент-владелец** | Process Agent + `kru-inspection-expert` |
| **Стандарт** | O'z DSt 1987:2018 §6.5.1 |

**Функции:**
- Полный жизненный цикл ревизии (Черновик → Исполнен → Архивный)
- Автоматический анализ транзакций на нарушения
- Управление составом ревизионной комиссии
- Контроль сроков через `deadline-watchdog`
- Интеграция с казначейством для получения финансовых данных

**Статусная модель:**
```
Черновик → На проверке → Утверждён → Исполнен → Архивный
              ↑                |
              └────────────────┘ (возврат на доработку)
```

**Входные данные:** Данные из ПС-01 (приказ), транзакции из казначейства
**Выходные данные:** Акт ревизии, реестр нарушений → ПС-03, ПС-04
**Связанные подсистемы:** ПС-01, ПС-03, ПС-05, ПС-06

---

## ПС-03: Учёт нарушений и решений

| Атрибут | Значение |
|:---|:---|
| **Агент-владелец** | Process Agent |
| **Стандарт** | O'z DSt 1987:2018 §6.5.2 |

**Функции:**
- Реестр выявленных нарушений с классификацией (финансовые, процедурные, документарные)
- Расчёт и учёт сумм ущерба в UZS
- Формирование решений по нарушениям
- Контроль исполнения решений (устранение нарушений)
- Отслеживание статусов: Зафиксировано → Решение принято → Исполнено → Закрыто

**Входные данные:** Результат анализа из ПС-02
**Выходные данные:** Статистика нарушений → ПС-04 (отчётность)
**Связанные подсистемы:** ПС-02, ПС-04, ПС-06

---

## ПС-04: Отчётность и аналитика

| Атрибут | Значение |
|:---|:---|
| **Агент-владелец** | Process Agent + `automated-reporting` |
| **Стандарт** | O'z DSt 1987:2018 §6.5.2 |

**Функции:**
- Генерация Актов ревизии (PDF, XLSX) по официальным шаблонам
- Сводные отчёты по нарушениям за период
- KPI-дашборд деятельности КРУ
- Годовые аналитические отчёты для руководства
- Отправка готовых актов на Единый портал госуслуг

**Форматы выходных документов:** PDF, XLSX, (перспективно: CSV)
**SLA генерации:** PDF ≤ 10с, XLSX ≤ 5с
**Связанные подсистемы:** ПС-02, ПС-03, ПС-05 (отправка на портал)

## ПС-05: Интеграционная шина

| Атрибут | Значение |
|:---|:---|
| **Агент-владелец** | Integration Agent + `external-integration` |
| **Стандарт** | O'z DSt 1987:2018 §6.5.4 |

**Внешние системы и протоколы:**

| Система | Протокол | Направление | Данные |
|:---|:---|:---|:---|
| Казначейство (Минфин) | SOAP/HTTPS | Входящий | Финансовые транзакции |
| Единый портал (Минюст) | REST/HTTPS | Исходящий | Акты ревизий |
| Налоговая служба | REST/HTTPS | Двунаправленный | Данные налогоплательщиков |
| ESI | OAuth2/OIDC | Входящий | Аутентификация |

**Принципы:** Идемпотентность, retry с backoff, TLS 1.2+, 100% логирование
**Связанные подсистемы:** Все (единая точка внешних коммуникаций)

## ПС-06: Безопасность и управление доступом

| Атрибут | Значение |
|:---|:---|
| **Агент-владелец** | Security Agent |
| **Стандарт** | O'z DSt 2814:2019; O'z DSt 3532:2021 |

**Функции:**
- RBAC — ролевая модель доступа (матрица: [RBAC_Matrix.md](../../05_Agents_и_Skills/Матрица_связей/RBAC_Matrix.md))
- Аутентификация пользователей (логин/пароль + интеграция с ESI)
- Маскирование ПИНФЛ во всех представлениях
- Журнал аудита всех действий (хранение 5 лет)
- Детекция аномалий и блокировка угроз
- Fail-closed: при недоступности — запрет всех операций

**Связанные подсистемы:** Все (является cross-cutting concern)

## ПС-07: Мониторинг и администрирование

| Атрибут | Значение |
|:---|:---|
| **Агент-владелец** | Monitoring Agent |
| **Стандарт** | O'z DSt 1987:2018 §4.4 — Требования к надёжности |

**Функции:**
- Сбор метрик всех подсистем (API latency, доступность, очереди)
- Алертинг при отклонении от SLA (WARNING/CRITICAL)
- Управление пользователями и ролями (ADMIN-интерфейс)
- Резервное копирование по расписанию
- Управление справочниками (НСИ)

**Целевые SLA:**

| Метрика | Цель |
|:---|:---|
| Доступность системы | ≥ 99.5% |
| API latency (p95) | ≤ 200 мс |
| RTO при сбое | ≤ 30 минут |
