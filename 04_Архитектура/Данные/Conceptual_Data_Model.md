## 2. Описание ключевых реестров

### 2.1. Реестр объектов (Units)

Хранит иерархию подразделений Министерства обороны.

* **НПА**: Используется для формирования плана (п. 29 Положения).

### 2.2. Реестр нарушений (Violations)

Центральное хранилище данных о выявленных недостатках. Каждое нарушение обязательно привязано к классификатору (п. 120 Положения).

* **Типы**: Хищение, Недостача, Незаконные выплаты, Утеря имущества МТО.

### 2.3. Реестр предписаний и решений (Decisions)

Учет юридически значимых действий КРУ.

* **SLA**: Система автоматически подсвечивает записи, по которым срок исполнения (10 дней) превышен (п. 135 Положения).

## 3. Требования к хранению (Data Policy)

* **Локальность**: Все данные хранятся на сервере в закрытом контуре.

* **История**: Хранение истории изменений (Versioning) для каждой записи нарушения.

* **Архивация**: Автоматический перенос завершенных дел в архивный сегмент через 3 года.
