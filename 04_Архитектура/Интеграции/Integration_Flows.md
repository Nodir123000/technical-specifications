# ПРИЛОЖЕНИЕ Ж — СПЕЦИФИКАЦИИ ИНТЕГРАЦИОННЫХ ПОТОКОВ

Автоматизированная Информационная Система Контрольно-Ревизионного Управления
Министерства Обороны Республики Узбекистан (АИС КРУ МО РУЗ)

---

## Ж.1. ОБЩИЕ ПОЛОЖЕНИЯ

### Ж.1.1. Назначение

Настоящее приложение описывает интеграционные потоки АИС КРУ МО РУЗ с внешними информационными системами, включая спецификации API, форматы данных, SLA и механизмы обработки ошибок.

### Ж.1.2. Перечень внешних систем

| № | Система | Тип | Статус интеграции | Приоритет |
|---|---------|------|------------------|----------|
| 1 | **E-XAT** | Электронный документооборот | Планируется | Критический |
| 2 | **Единое окно** | Межведомственное взаимодействие | Планируется | Высокий |
| 3 | **НПБ** | Нормативная правовая база | Планируется | Средний |
| 4 | **ГосИС «Единый реестр»** | Верификация физлиц | Планируется | Высокий |

## Ж.2. ИНТЕГРАЦИЯ С E-XAT

### Ж.2.1. Описание системы

| Параметр | Значение |
|----------|---------|
| **Полное наименование** | E-XAT — Электронный документооборот |
| **Организация** | Министерство обороны РУз |
| **Протокол** | REST API (HTTPS) |
| **Формат данных** | JSON |
| **Аутентификация** | mTLS (X.509) |

### Ж.2.2. API-спецификация E-XAT

#### Ж.2.2.1. Получение входящих документов

| Параметр | Значение |
|----------|---------|
| **Метод** | GET |
| **Endpoint** | `/api/v1/incoming-documents` |
| **Авторизация** | Bearer Token |

**Query Parameters:**

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|---------|
| page | integer | Нет | Номер страницы (default: 1) |
| pageSize | integer | Нет | Размер страницы (default: 20, max: 100) |
| dateFrom | date | Нет | Дата начала периода |
| dateTo | date | Нет | Дата окончания периода |
| status | string | Нет | Статус документа |

**Response (200 OK):**

```json
{
  "data": [
    {
      "id": "uuid",
      "documentNumber": "string",
      "documentDate": "date",
      "documentType": "string",
      "subject": "string",
      "sender": {
        "organization": "string",
        "department": "string",
        "fio": "string"
      },
      "status": "received|processed|archived",
      "attachments": [
        {
          "id": "uuid",
          "fileName": "string",
          "fileType": "string",
          "fileSize": "integer",
          "downloadUrl": "string"
        }
      ],
      "receivedAt": "datetime"
    }
  ],
  "pagination": {
    "page": "integer",
    "pageSize": "integer",
    "totalItems": "integer",
    "totalPages": "integer"
  }
}
```

#### Ж.2.2.2. Отправка документа (приказ, акт)

| Параметр | Значение |
|----------|---------|
| **Метод** | POST |
| **Endpoint** | `/api/v1/outgoing-documents` |
| **Авторизация** | Bearer Token |
| **Content-Type** | multipart/form-data |

**Request Body:**

| Поле | Тип | Обязательный | Описание |
|------|-----|-------------|---------|
| documentType | string | Да | Тип документа (order/report) |
| subject | string | Да | Тема документа |
| body | string | Да | Текст документа (HTML) |
| recipientId | string | Да | Получатель |
| attachments[] | file | Нет | Файлы |
| metadata | object | Нет | Доп. метаданные |

**Response (201 Created):**

```json
{
  "id": "uuid",
  "documentNumber": "string",
  "status": "submitted",
  "submittedAt": "datetime"
}
```

### Ж.2.3. SLA E-XAT

| Метрика | Значение | Описание |
|---------|---------|---------|
| **Availability** | 99.5% | Доступность системы |
| **Response Time (p95)** | ≤ 3 сек | Время отклика API |
| **Throughput** | 100 req/min | Пропускная способность |
| **Timeout** | 30 сек | Таймаут операции |

### Ж.2.4. Обработка ошибок E-XAT

| Код ошибки | HTTP Status | Описание | Действие |
|------------|------------|---------|---------|
| E-XAT-001 | 400 | Некорректный формат данных | Проверить тело запроса |
| E-XAT-002 | 401 | Ошибка аутентификации | Проверить сертификат |
| E-XAT-003 | 403 | Доступ запрещён | Проверить права |
| E-XAT-004 | 404 | Документ не найден | Проверить ID |
| E-XAT-005 | 429 | Превышен лимит запросов | Retry с backoff |
| E-XAT-006 | 500 | Внутренняя ошибка сервера | Retry через 5 мин |

## Ж.6. ОБЩИЕ ТРЕБОВАНИЯ К ИНТЕГРАЦИЯМ

### Ж.6.1. Безопасность

| Требование | Реализация |
|-----------|-----------|
| **Транспорт** | TLS 1.3 only (без TLS 1.2 для внешних систем) |
| **Аутентификация** | mTLS для E-XAT, OAuth 2.0 + JWT для Единого окна |
| **Сертификаты** | X.509 v3, срок действия ≤ 2 лет, ротация ежегодно |
| **Шифрование данных** | AES-256 для чувствительных данных (ПИНФЛ, ПДн) |
| **Логирование** | Все запросы/ответы в audit_log |
| **WAF** | Обязательно для всех внешних API |
| **IP whitelist** | Только разрешённые IP для интеграций |

### Ж.6.2. Обработка ошибок и Retry

#### Стратегия Retry

| Параметр | Значение |
|----------|---------|
| **Max retries** | 3 |
| **Initial delay** | 1 сек |
| **Backoff** | Exponential (x2) |
| **Max delay** | 30 сек |
| **Retry on** | 429, 500, 502, 503, 504 |

#### Схема обработки ошибок (обновлённая)

```
Запрос к внешней системе
        │
        ▼
┌─────────────────┐
│  Успешно (2xx)? │
└────────┬────────┘
         │
    ┌────┴────┐
    │Да       │Нет
    ▼         ▼
┌────────┐  ┌─────────────────┐
│Возврат │  │ Retries < 3?    │
│данных  │  └────────┬────────┘
└────────┘       ┌────┴────┐
            ┌────┴────┐    │Нет
            │Да        │    ▼
            ▼         ▼  ┌─────────────────┐
      ┌─────────┐   ┌──────────────┐│ Запись в журнал │
      │ Wait + │   │ Запись в     ││ ошибок + Alert  │
      │ Retry  │   │ fail queue   │└─────────────────┘
      │(exp+backoff)│ (Kafka/RabbitMQ)│
      └─────────┘   └──────────────┘

Примечание: fail queue хранит сообщения для повторной обработки вручную.
План отказоустойчивости обязателен для каждой интеграции.
```

### Ж.6.3. Форматы данных

| Формат | Использование | Версия |
|--------|-------------|--------|
