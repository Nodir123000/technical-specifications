# ПРИЛОЖЕНИЕ Г — АРХИТЕКТУРА СИСТЕМЫ (C4 MODEL)

Автоматизированная Информационная Система Контрольно-Ревизионного Управления
Министерства Обороны Республики Узбекистан (АИС КРУ МО РУЗ)

---

## Г.1. ОБЩИЕ ПОЛОЖЕНИЯ

### Г.1.1. Назначение

Настоящее приложение описывает архитектуру АИС КРУ МО РУЗ с использованием модели C4 (Context, Container, Component, Code) для визуализации структуры системы на различных уровнях детализации.

### Г.1.2. Уровни абстракции C4

| Уровень | Назначение | Аудитория |
|---------|-----------|-----------|
| **Context (C4-1)** | Общий обзор системы и её окружения | Менеджеры, заинтересованные стороны |
| **Container (C4-2)** | Контейнеры (приложения, сервисы) | Разработчики, DevOps |
| **Component (C4-3)** | Компоненты внутри контейнеров | Разработчики |
| **Code (C4-4)** | Детали реализации (опционально) | Разработчики |

|-----|---------|---------|----------|
| 1 | Пользователь | Ввод логина и пароля | |
| 2 | Web App | Отправка POST /api/auth/login | Запрос credentials |
| 3 | AuthService | Валидация данных | Проверка формата |
| 4 | UserRepository | Поиск пользователя в БД | Запрос SELECT |
| 5 | UserRepository | Проверка хеша пароля | bcrypt.compare |

| 9 | Web App | Сохранение токенов | localStorage/cookie |

| 12 | **audit_log** | Журнал аудита | id, user_id, action, table_name | |

| 2 | quarterly_plans | включает | monthly_plans | 1:N |
