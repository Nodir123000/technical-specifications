## Г.5. ДИАГРАММЫ ПОСЛЕДОВАТЕЛЬНОСТИ (SEQUENCE)

### Г.5.1. Sequence: Локальная авторизация (PIN/Token)

1. **Пользователь** → **Web App**: Ввод логина и пароля.
2. **Web App** → **Security Agent**: Проверка в локальной БД.
3. **Security Agent** → **Web App**: Запрос 2nd factor (PIN-код).
4. **Пользователь** → **Web App**: Ввод 4-значного PIN-кода.
5. **Web App** → **Security Agent**: Валидация PIN + проверка серийного номера носителя (если подключен).
6. **Security Agent** → **Web App**: Успешный вход, выдача сессионного токена.

### Г.5.2. Sequence: Импорт данных из Excel

1. **Инспектор** → **Web App**: Выбор файла "Выписка_Казначейства.xlsx".
2. **Web App** → **Security Agent**: Проверка флага `security_scan_status` (АРМ-фильтр).
3. **Security Agent** → **Integration Agent**: Разрешение на парсинг и проверку SHA-256.
4. **Integration Agent** → **local-excel-import**: Чтение строк, валидация шаблона.
5. **Integration Agent** → **Process Agent**: Передача JSON-массива данных.
6. **Process Agent** → **БД**: INSERT транзакций с привязкой к ревизии.
