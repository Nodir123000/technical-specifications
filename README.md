# 📁 Технические задания: Методология Слоев

Данный репозиторий/каталог организован по принципу разделения ответственности (Layered Logic), что обеспечивает прослеживаемость от законов до программного кода.

## Структура каталогов

### 🏛️ 01_Нормативка (Основа)
Фундамент системы. Сюда помещаются государственные стандарты (O'z DSt, ГОСТ), международные стандарты (ISO) и внутренние регламенты заказчика.
*   **Цель:** Обеспечить юридическую и техническую легитимность системы.

### 📥 02_Документы_заказчика (Сырье)
Неизменяемое хранилище исходных требований от заказчика: письма, протоколы, наброски ТЗ, описание бизнес-процессов "как есть".
*   **Правило:** Файлы в этой папке не редактируются.

### 🧠 03_Анализ (Смысл)
Результат обработки "хотелок" заказчика. Здесь происходит декомпозиция требований, Gap-анализ, оценка рисков и ограничений.

### ⚙️ 04_Архитектура (Мозг)
Техническое описание системы: общая архитектура, схемы баз данных, интеграционные интерфейсы, диаграммы компонентов.

### 🤖 05_Agents_и_Skills (Автономность)
Описание логики ИИ-агентов и их навыков (Skills). Определение сценариев взаимодействия агентов с пользователями и друг с другом.

### 📜 06_Техническое_задание (Финал)
Сборка итогового документа согласно O'z DSt 1987:2018. Содержит черновики, версии для согласования и финальную версию с подписями.

### 🧪 07_Тестирование_и_приемка
Методика и программа испытаний (ПМИ), тест-кейсы, сценарии приемки и акты выполненных работ.

### 📚 08_Документация
Руководства пользователя, администратора и проектная документация для передачи в эксплуатацию.

### 🔗 09_Трассировка (Контроль)
Матрица соответствия требований и функций. Гарантирует, что каждое требование заказчика реализовано, и в системе нет "лишних" функций.
