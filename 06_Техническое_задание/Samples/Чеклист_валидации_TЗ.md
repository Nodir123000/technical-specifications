# 🧪 Чеклист валидатора технического задания

Автоматическая проверка полноты и корректности ТЗ перед сдачей.

---

## ✅ Раздел 1: Общие сведения

* [ ] Наименование системы указано

* [ ] Шифр (обозначение) присвоен

* [ ] Заказчик: наименование, адрес, контактное лицо

* [ ] Разработчик: наименование, адрес, контактное лицо

* [ ] Исполнитель: наименование, адрес, контактное лицо (если есть)

---

## ✅ Раздел 2: Назначение и цели

* [ ] Назначение системы описано (минимум 3 предложения)

* [ ] Цели создания — минимум 3 цели

* [ ] Цели имеют измеримые результаты (таблица с метриками)

---

## ✅ Раздел 3: Характеристика объектов автоматизации

* [ ] Перечень организаций (минимум 3 типа)

* [ ] Количество автоматизированных рабочих мест

* [ ] Таблица АРМ с функциями

---

## ✅ Раздел 4: Требования к системе

### 4.1 Функциональные требования

* [ ] Минимум 3 подраздела (4.1.1, 4.1.2, 4.1.3)

* [ ] Каждое требование имеет ID (F-XX)

* [ ] Каждое требование имеет приоритет (Высокий/Средний/Низкий)

* [ ] Каждое требование привязано к навыку (Skill)

* [ ] Указана реализация: Agent + Skill

### 4.2 Интерфейс

* [ ] Требования к пользовательскому интерфейсу

* [ ] Локализация (RU, uz-Latn, uz-Cyrl)

* [ ] Требования к программному интерфейсу (API)

### 4.3 Безопасность

* [ ] Маскирование ПИНФЛ описано

* [ ] RBAC определён

* [ ] Аудит-лог задокументирован

### 4.4 Надёжность

* [ ] Доступность (≥ 99.5%)

* [ ] Время восстановления (≤ 30 мин)

* [ ] Автосохранение

### 4.5 Архитектура

* [ ] Технологический стек определён

* [ ] Структура базы данных описана

* [ ] Интеграционные интерфейсы определены

---

## ✅ Раздел 5: Состав и содержание работ

* [ ] Минимум 5 этапов

* [ ] Для каждого этапа: наименование, сроки, результат

* [ ] Timeline представлен в таблице

---

## ✅ Раздел 6: Порядок контроля и приёмки

* [ ] Определены виды испытаний

* [ ] Определены критерии приёмки (минимум 3)

* [ ] Указаны методы проверки для каждого критерия

## ✅ Раздел 7: Требования к подготовке объекта

* [ ] Требования к помещениям

* [ ] Требования к оборудованию (клиентскому)

* [ ] Требования к сетевой инфраструктуре

---

## ✅ Раздел 8: Документирование

* [ ] Перечень документации

* [ ] Формат документов (PDF, DOCX, MD)

* [ ] Сроки предоставления

---

## ✅ Раздел 9: Источники разработки

* [ ] Нормативные документы (O'z DSt 1987:2018 и др.)

* [ ] Технические требования заказчика

* [ ] Проектная документация (если есть)
