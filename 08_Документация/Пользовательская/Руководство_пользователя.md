## 1. Введение

### 1.1 Назначение документа

Настоящее Руководство пользователя предназначено для сотрудников Контрольно-Ревизионного Управления Министерства Обороны Республики Узбекистан, работающих с АИС КРУ МО РУЗ.

### 1.2 Аудитория

| Роль | Разделы |
|:---|:---|
| Инспектор-ревизор (INSPECTOR) | Все разделы |
| Главный инспектор (CHIEF_INSPECTOR) | Разделы 2–6 |
| Наблюдатель (VIEWER) | Разделы 2–3 |

### 1.3 Требования к рабочему месту

- Браузер: Google Chrome ≥ 110, Mozilla Firefox ≥ 110, Edge ≥ 110
- Разрешение экрана: ≥ 1280×768
- Интернет-соединение: ≥ 5 Мбит/с
- Электронная цифровая подпись (ЭЦП) — для утверждения документов

## 2. Начало работы

### 2.1 Вход в систему

1. Откройте браузер и перейдите по адресу системы
2. Введите **логин** (присвоен администратором) и **пароль**
3. При первом входе система потребует сменить пароль
4. Нажмите **«Войти»**

> ⚠️ **Важно:** Сессия автоматически завершается через **30 минут** бездействия. Несохранённые данные будут потеряны.

### 2.2 Смена языка интерфейса

Система поддерживает три языка. Переключение — в правом верхнем углу:

| Пункт меню | Язык |
|:---|:---|
| РУС | Русский |
| O'Z | Узбекский (латиница) |
| ЎЗ | Узбекский (кириллица) |

### 2.3 Главное меню

| Раздел | Описание | Роль |
|:---|:---|:---|
| Планирование | Годовые, квартальные, месячные планы | INSPECTOR+ |
| Ревизии | Создание и ведение ревизий | INSPECTOR+ |
| Нарушения | Реестр выявленных нарушений | INSPECTOR+ |
| Отчёты | Генерация и просмотр отчётов | INSPECTOR+ |
| Справочники | Организации, классификаторы | Только чтение (VIEWER) |
| Администрирование | Управление пользователями | Только ADMIN |

## 3. Работа с ревизиями

### 3.1 Создание новой ревизии

1. В меню выберите **«Ревизии»** → **«Создать»**
2. Заполните обязательные поля:
   - **Организация** — выберите из справочника
   - **Период ревизии** — укажите даты «от» и «до»
   - **Тип ревизии** — полная / тематическая / частичная
   - **Состав комиссии** — назначьте инспекторов
3. Нажмите **«Сохранить черновик»**

> ℹ️ Система автоматически запросит финансовые данные организации из системы казначейства. Это может занять до 30 секунд.

### 3.2 Регистрация нарушения

1. Откройте ревизию → вкладка **«Нарушения»** → **«Добавить»**
2. Заполните:
   - **Тип нарушения** — финансовое / процедурное / документарное
   - **Сумма ущерба (UZS)**
   - **Ссылка на документ** — номер и дата
   - **Описание** — краткое изложение нарушения
3. Нажмите **«Зарегистрировать»**

> ℹ️ Система автоматически оценивает нарушение. При недостаточности данных — запросит подтверждение у главного инспектора.

### 3.3 Формирование Акта ревизии

> Доступно только для роли **CHIEF_INSPECTOR**

1. Откройте завершённую ревизию → **«Сформировать Акт»**
2. Выберите формат: **PDF**, **Excel** или **оба**
3. Дождитесь генерации (до 30 секунд)
4. Проверьте документ, нажмите **«Утвердить»** или **«Вернуть на доработку»**

## 4. Работа с отчётами

### 4.1 Генерация отчёта

1. Меню → **«Отчёты»** → **«Создать отчёт»**
2. Выберите тип: Акт ревизии / Сводный отчёт / KPI-дашборд / Годовой отчёт
3. Укажите период и организации
4. Нажмите **«Сгенерировать»**

### 4.2 Экспорт

Готовые отчёты доступны для скачивания:
- **PDF** — для печати и подписания
- **XLSX** — для редактирования

> ⚠️ Функция экспорта недоступна для роли **VIEWER**.

## 6. Часто задаваемые вопросы (FAQ)

**В: Я не вижу кнопку "Утвердить" — почему?**
О: Эта кнопка доступна только для роли CHIEF_INSPECTOR. Обратитесь к администратору системы.

**В: Система пишет "Внешний сервис недоступен" — что делать?**
О: Данные казначейства временно недоступны. Ревизия сохранена. Данные синхронизируются автоматически. Если проблема не исчезнет через 2 часа — обратитесь к администратору.

**В: Как скрыт ПИНФЛ — это нормально?**
О: Да. Система отображает ПИНФЛ в маскированном виде (123\*\*\*\*0123) в соответствии с O'z DSt 3532:2021.

**В: Моя сессия истекла и я потерял данные — как восстановить?**
О: Черновик формы автосохраняется каждые 2 минуты. При повторном входе система предложит восстановить несохранённые данные.
